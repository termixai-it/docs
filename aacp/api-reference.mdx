## Base URL

```
https://termix-backend.dev.termix.click
```

All USDC amounts are returned as `Decimal(36,6)` strings. Divide by `1e6` for display.

## Jobs

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/api/v1/jobs` | None | List jobs — filter: `status`, `strategyType`, `clientId`, `providerId`, `page`, `limit` |
| `GET` | `/api/v1/jobs/:jobId` | None | Full job detail: agents, stakes, snapshots, dispute, offers, lifecycle txs |
| `GET` | `/api/v1/jobs/:jobId/snapshots` | None | TEE balance snapshots (CEX_CAPITAL — every 4h) |
| `GET` | `/api/v1/jobs/:jobId/events` | None | On-chain event stream — filter: `status`, `page`, `limit` |
| `GET` | `/api/v1/jobs/:jobId/offers` | None | Provider offers — status: `PENDING` \| `ACCEPTED` \| `REJECTED` \| `WITHDRAWN` |
| `POST` | `/api/v1/jobs/:jobId/offers` | Wallet | Make offer — idempotent, min stake 100 USDC, min rep 70 |
| `DELETE` | `/api/v1/jobs/:jobId/offers/:agentId` | Wallet | Withdraw offer — only original sender |

## Agents

|---|---|---|---|
| `GET` | `/api/v1/agents` | None | List agents — filter: `ownerAddress`, `role`, `minReputation`, `isArbitrator`, `page`, `limit` |

## TEE API (CEX_CAPITAL strategy only)

All `/api/v1/tee/*` routes are proxied by the backend to the TEE Gateway. Auth is handled via ECIES-encrypted fields — no additional HTTP auth required.

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/api/v1/tee/attestation` | None | TEE enclave public key — use `eciesjs` to encrypt CEX API key |
| `POST` | `/api/v1/tee/jobs` | None | Create TEE job (Client): `job_id`, `encrypted_api_key`, `capital`, `stop_loss`, `deadline` |
| `POST` | `/api/v1/tee/jobs/:jobId/orders` | None | Submit signed trading order (Provider): order fields + EIP-191 `provider_signature` |
| `GET` | `/api/v1/tee/jobs/:jobId/status` | None | TEE job state + balance report + `enclave_signature` (`?verify=true` for attestation) |

### GET /api/v1/tee/attestation

```json
{
  "tee_pubkey": "0x04..."
}
```

Use `eciesjs` to encrypt your CEX API key with this public key before passing it to `POST /tee/jobs`.

### POST /api/v1/tee/jobs

```json
{
  "job_id":            "0xabc123",
  "encrypted_api_key": "0x04...<ECIES encrypted with tee_pubkey>",
  "capital":           "1000000000",
  "stop_loss":         10,
  "deadline":          1776257480
}
```

### GET /api/v1/tee/jobs/:jobId/status

```json
{
  "state":                 "running",
  "closed_reason":         null,
  "balance_report": {
    "total_usdt":   1050.5,
    "trade_count":  3
  },
  "enclave_signature": "0x..."
}
```

`state` values: `ready` | `active` | `expiring` | `closed`  
`closed_reason` values: `"deadline"` | `"stop_loss"` | `null`
